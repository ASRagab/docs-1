Nitric Functions are a type of [service](./overview) that offers a provider independent way to develop Function as a Service (FaaS) applications, which may be deployed to Serverless Runtimes such as AWS Lambda and Google CloudRun.

## Create Functions

Functions are created the same way as [services](./overview) but use specialized templates for easy scaffolding and maintenance.

These templates are available in the Nitric [function templates](https://github.com/nitrictech/function-templates) repository. This can be retrieved with the Nitric CLI by using.

`nitric templates:repos:add function`

You can then add a function to your existing application using:

`nitric add:service`

The newly available function templates should be visible in the list to select from `function/<template-name>`.

Currently available function templates are:

* golang15
* python37
* nodejs12
* nodets12
* java11

## Function Entrypoint

Functions are simple programs that host a server to receive requests and return responses, via the Nitric Membrane.

> see [Overview](./overview)

The Nitric SDKs provide methods that accept a function and then run that function as a server. Find examples for available SDKs below:

## Function Requests & Responses

Nitric functions provide a request and response model that is completely trigger agnostic. Whether receiving an event from a topic or an HTTP Request, you will always receive the same kind of request object and return the same kind of response object.

The interfaces for each of these are idiomatic to the runtime they were written for, for more information and examples on using them
checkout out our github [repository](https://github.com/nitrictech) for the SDK for your favorite language.
