### Task IDs

Every task that's produced can be uniquely identified so that receivers can be idempotent. To assist with this, Nitric Tasks include an ID property which should contain the unique ID for that task. By default, the Nitric Membrane will automatically generate this ID as a v4 UUID for every published task if it doesn't already contain an ID. Custom ID creation should be implemented by your code to ensure that duplicate queue items contain the same ID value.

### Failed Tasks

For a variety of possible reasons, tasks may fail to be sent to a queue. To combat this, after sending tasks to a queue, the failed tasks will be returned. The failed tasks come packaged with the original task and the reason it failed. This lets you isolate problematic tasks, determine why they didn't process, and then resend them if possible.
