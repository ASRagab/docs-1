Presigned URLS can be used with the Nitric API to give temporary read or write access to an object in a bucket. This URL can be distributed to anyone, without them requiring permissions or a google account.

<CodeExample lang={gcp}>

To set up presigned-urls for GCP you first need to set up a service account that has access to the bucket. To do this, navigate to IAM & Admin > Service Accounts and press the big blue buttont hat says "Create Service Account".

Once the service account is created, go into the service account and go to the keys tab. Then press Add Key > Create new key > JSON.

This will download a JSON file that can be used to build the presigned url. Set the SERVICE_ACCOUNT_LOCATION environment variable to point
to the location of the JSON file.

### Limitations

- The maximum expiry for a gcp presigned url is 604800 seconds (7 days)
